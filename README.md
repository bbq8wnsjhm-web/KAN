# Сайт клуба

Этот проект сейчас удобно воспринимать как две рабочие зоны:

- `C:\Users\ASUS\Desktop\сайт` — готовый статический сайт.
- `C:\Users\ASUS\Desktop\сайт\моушнсайт` — мастерская с исходниками, черновиками, шрифтами и текстами.

Чтобы нам было проще работать вместе, в проект добавлены:

- `docs/asset-map.md` — какие медиа сейчас реально подключены к страницам сайта.
- `docs/design-rules.md` — визуальные и интерактивные правила, включая ориентир на `monopo.london`.
- `docs/layout-rules.md` — строгие правила сетки, композиции, иерархии текста и responsive-верстки.
- `docs/workflow.md` — правила раскладки материалов и рабочий процесс.
- `tools/start-preview.ps1` — локальный запуск сайта через Python.
- `.editorconfig` — единая кодировка UTF-8 и базовые правила для текстовых файлов.
- `.gitignore` — игнорирование служебных логов и резервных данных из `моушнсайт`.

## Быстрый старт

Из корня проекта:

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\start-preview.ps1
```

После запуска сайт будет доступен по адресу `http://localhost:8000`.

## Короткое правило работы

- Всё, что реально подключено в HTML, держим в `media/`.
- Всё, что является исходником, наброском или мастер-материалом для будущих сцен, держим в `моушнсайт/`.
- Перед добавлением нового видео в сайт проверяем, что оно относится к конкретной странице и зафиксировано в `docs/asset-map.md`.
- Новые тексты лучше сначала собирать отдельно, а потом переносить в HTML, чтобы не терять формулировки при быстрых правках.
- При визуальных правках и новых сценах сверяемся с `docs/design-rules.md`, чтобы не потерять аккуратность и цельность интерфейса.
- При пересборке композиции и верстки сверяемся с `docs/layout-rules.md`: колонками, ритмом, иерархией текста и поведением навигации.
- Для live-сайта стандартный формат анимации — `mp4`. Формат `mov` считаем мастер- или промежуточным форматом и по возможности не подключаем напрямую в HTML.
- Все фоновые и сценические анимации на сайте по умолчанию должны быть зациклены и запускаться автоматически.
- Для autoplay-видео в сайте стандартный набор атрибутов: `autoplay`, `muted`, `loop`, `playsinline`, а также `poster` при необходимости.
- Базовый UI-шрифт сайта — локальный PT Root UI из `media/fonts/pt-root-ui/`, а не внешний CDN.

## Что дальше

Следующий спокойный шаг — привести `моушнсайт` к более строгим ролям папок: источник, экспорт, архив, текст, шрифт. Предлагаемая схема уже описана в `docs/workflow.md`.
