# Рабочий процесс

Сейчас корень проекта уже работает как боевой сайт, а `моушнсайт` больше похож на общую мастерскую. Для совместной работы это нормально, но лучше заранее договориться о ролях папок.

## Базовый принцип

- `media/` — только то, что реально используется сайтом прямо сейчас.
- `моушнсайт/` — всё, что помогает производить контент: исходники, эксперименты, черновые рендеры, текст, шрифты.
- Визуальные решения и интерактивность сверяем с `docs/design-rules.md`, чтобы у сайта был устойчивый характер, а не набор разрозненных эффектов.
- Правила сетки, композиции и editorial-верстки сверяем с `docs/layout-rules.md`.

## Форматы анимации

- Для сайта основным форматом анимации считаем `mp4`.
- `mov` используем как исходный, монтажный или архивный формат внутри `моушнсайт/`.
- Если ролик должен попасть в `media/` и использоваться в HTML, целевой формат по умолчанию — `mp4`.
- Подключать `mov` в live-страницы нежелательно, потому что это чаще создаёт проблемы совместимости и предсказуемости в браузерах.
- Если нужен мастер без потери качества, его можно хранить как `mov`, но в сайт всё равно лучше отдавать отдельный `mp4`.
- Для конвертации мастер-роликов используем `tools/convert-mov-to-mp4.ps1`, который опирается на локально установленный FFmpeg.

## Стандарт для видео в сайте

- Все фоновые и сценические анимации в live-сайте по умолчанию должны быть зациклены.
- Все такие анимации по умолчанию должны запускаться автоматически.
- Чтобы autoplay работал стабильно в браузерах, видео должно быть без звука и с атрибутом `muted`.
- Для live-видео используем стандартный набор атрибутов: `autoplay`, `muted`, `loop`, `playsinline`.
- Если важен аккуратный первый кадр, добавляем `poster`.

## Шрифты

- Локальные мастер-шрифты держим в `моушнсайт/fonts/` как мастер-источник.
- В live-сайт подключаем web-копии из `media/fonts/` через `@font-face`.
- Внешние загрузки шрифтов для боевой версии не используем.
- Текущий основной UI-шрифт проекта — PT Root UI.
- Старые шрифты, выведенные из live-сайта, складываем в `моушнсайт/archive/fonts/`.

## Короткий чек перед вставкой нового видео

- Есть ли мастер-файл в `моушнсайт/`.
- Подготовлена ли web-версия в `mp4`.
- Зациклено ли видео без заметного шва.
- Будет ли оно играть нормально без звука.
- Нужен ли `poster`, чтобы не было пустого кадра при загрузке.
- Добавлен ли ролик в `docs/asset-map.md`, если он стал live-ассетом.

## Как трактовать текущие папки

- `моушнсайт/анимация` — хорошее место для экспортов, которые уже близки к использованию.
- `моушнсайт/текст` — место для черновиков текста перед переносом в HTML.
- `моушнсайт/fonts` — локальные мастер-шрифты проекта.
- `моушнсайт/свалка` — временный буфер. Лучше не хранить там финальные вещи надолго.
- `моушнсайт/archive/legacy-live` — заменённые или снятые с live мастер-ролики и старые версии ассетов.

## Целевая схема

```text
моушнсайт/
  source/
  export/
    draft/
    approved/
  copy/
  fonts/
  archive/
```

Смысл такой:

- `source/` — TouchDesigner, исходные композиции, проектные файлы.
- `export/draft/` — промежуточные рендеры для просмотра.
- `export/approved/` — то, что уже можно кандидатить в `media/`.
- `copy/` — тексты по страницам.
- `fonts/` — шрифты и лицензии.
- `archive/` — старые варианты, которые не нужны в live, но полезно сохранить.

## Правила именования

Лучше держаться формата:

- `main-bg-v01.mp4`
- `kto-bg-v03.mp4`
- `find-bg-v02.mp4`
- `what-cube-poster-v01.jpg`

Если файл уже ушёл в боевой сайт, в `media/` можно упрощать имя до стабильного:

- `main-bg.mp4`
- `kto-bg.mp4`
- `find-bg.mp4`
- `what-bg.mp4`
